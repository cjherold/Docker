Deployment Models

The deployment models of cloud computing are based on where the hardware is relative to the user. The types of deployments are Public, Private, Hybrid, and Multi-Cloud. Cloud models can differ greatly, and one company's deployment might look completely different than another's.

Public

Public clouds are often run on shared resources as a service. Like Amazon's AWS, there are many different organizations using servers that are owned by Amazon. Although the hardware may be from the same place, the different organizations do not have access to each other's data. Some issues with this deployment model can include data ownership, security, and performance of a shared resource.

Private

Private clouds are usually run on an organization's own hardware. They can be very secure, and performance depends on the quality of the installed hardware and configuration. Private clouds can be expensive to build initially. Also they can be harder to scale early on than public clouds but there are many benefits to them in the long run.

Hybrid

Hybrid clouds are a combination of public and private deployments. An organization may keep more sensative data held in the private cloud and then use the public cloud for less sensative or cpu intensive tasks that they do not have the resources for. This provides the benefits of being able to scale fast while building up a private cloud network.

Multi-Cloud

Multi-Clouds contain a combination of different cloud models that are not all interconnected. One example of this could be a company that has a private network for employee payroll and an unconnected hybrid network that provides services for an app they own.
